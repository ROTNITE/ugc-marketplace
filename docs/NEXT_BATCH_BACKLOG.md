# Пакет 2: UX-флоу улучшения (Backlog)

## Основания
- Источник: `docs/UI_AUDIT.md`, `CONCEPT.md`, код страниц `src/app/**`.
- Ручной прогон UI через браузер в этом окружении не выполнялся: часть задач требует валидации на локальном запуске.

## Карта сценариев (сокращенно)

### Brand
1) Создать заказ → отправить на модерацию → дождаться одобрения.
2) Получить отклики/приглашения → выбрать исполнителя → открыть чат.
3) Пополнить эскроу → креатор сдаёт материалы → приёмка/правки.
4) Завершить сделку → оставить отзыв.

### Creator
1) Найти заказ → отправить отклик (или принять приглашение).
2) Получить решение → открыть чат → выполнить работу.
3) Отправить материалы → дождаться решения → правки при необходимости.
4) Получить выплату на баланс → запросить выплату → дождаться одобрения.
5) Оставить отзыв.

### Admin
1) Модерация заказов → approve/reject с причиной.
2) Верификация креаторов.
3) Выплаты: approve/reject запросов.
4) Споры/отмены и финансы (ledger/escrow).
